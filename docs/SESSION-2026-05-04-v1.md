### 7.5 Self-allow regex extension applied this session

To unblock memory file maintenance (where filenames like `feedback_never_read_secret_files.md` triggered the credential-file regex), the `is_self_path` function in `policy-block.sh` was extended with a regex matching `~/.claude/projects/<project>/memory/<file>.md`. This is host-portable (works for any user's `$HOME`).

The change applied is two edits in `policy-block.sh`:

Addition after `SELF_ALLOW_PREFIXES`:

```python
SELF_ALLOW_MEMORY_RE = re.compile(
    r'^' + re.escape(os.path.expanduser("~/.claude/projects/")) + r'[^/]+/memory/[^/]+\.md$'
)
```

Modified `is_self_path`:

```python
def is_self_path(p):
    if not p:
        return False
    try:
        ap = os.path.abspath(os.path.expanduser(p))
        if any(ap.startswith(prefix) for prefix in SELF_ALLOW_PREFIXES):
            return True
        if SELF_ALLOW_MEMORY_RE.match(ap):
            return True
        return False
    except Exception:
        return False
```

After the change: integrity check confirmed passing. `Read` of `~/.claude/projects/-root/memory/feedback_never_read_secret_files.md` now succeeds. `Bash` commands that contain the literal substring `secret_files` are still blocked because `policy-block.sh` deliberately does not apply self-allow to Bash command text; this is by design (a Bash command line that mentions a self-path should not bypass other secret-path triggers in the same line).

## 8. The opencode bridge

### 8.1 Purpose

Make opencode obey the same canonical security policy at `~/.claude/settings.json` and run the same hook scripts at `~/.claude/hooks/` that Claude Code runs natively. No duplication of regex, deny lists, or hook logic. One source of truth, two agents reading from it.

### 8.2 Bridge plugin file

Path: `$HOME/.config/opencode/plugin/claude-bridge.ts`

Approximately 320 lines. Implementation:

- Loads `~/.claude/settings.json` on plugin init (re-reads at session start to pick up edits).
- Parses `permissions.deny` rules with a `Tool(glob)` matcher; converts globs to regex (`**` → `.*`, `*` → `[^/]*`, escape other regex metas; anchored with leading `(?:.*/)?`).
- Maps opencode tool names to Claude tool names: `bash→Bash`, `read→Read`, `write→Write`, `edit→Edit`, `multiedit→Edit`, `patch→Edit`, `glob→Glob`, `grep→Grep`, `webfetch→WebFetch`, `websearch→WebSearch`, `task→Agent`, `todowrite→TaskUpdate`, `todoread→TaskList`. Unknown tools (mcp__*) pass through unchanged.
- Implements opencode plugin hooks:
  - `tool.execute.before`: evaluates deny rules for the mapped Claude tool name against the path-shaped argument; throws on match. Then runs each matching `PreToolUse` hook script with stdin JSON `{session_id, hook_event_name: "PreToolUse", tool_name, tool_input}`. Parses stdout JSON; if `permissionDecision` is `deny` or `ask`, throws with `permissionDecisionReason`. Surfaces `systemMessage` to stderr and via opencode toast event.
  - `tool.execute.after`: runs `PostToolUse` hooks with the tool response, surfaces messages.
  - `event`: on `session.created` (first time per sessionID), runs `SessionStart` hooks. On `session.deleted`, runs `SessionEnd` hooks.
- Module export: `export default { id: "claude-bridge", server: plugin }`. The `id` field is required by opencode for path-loaded plugins.
- Fail-closed posture: if `~/.claude/settings.json` is missing or unparseable, the plugin synthesizes a deny-everything settings object so the agent cannot operate without policy.
